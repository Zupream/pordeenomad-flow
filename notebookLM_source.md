# Pordeenomad Commission & Affiliate Flow — เอกสารสรุปสำหรับ NotebookLM

> เอกสารนี้รวบรวมโครงสร้างระบบ Commission ของ Pordeenomad (V3 — Full BStore Model) พร้อมตัวอย่างการคำนวณ

---

## 1. ภาพรวมระบบ

Pordeenomad เป็น **Marketplace** ที่ Seller ภายนอกมาลงขายสินค้า โดยมีระบบ Affiliate ให้คนมาแชร์ลิงก์ขายสินค้า

### บทบาทในระบบ
| บทบาท | หน้าที่ |
|--------|---------|
| **Buyer** | ซื้อสินค้าผ่าน Affiliate Link → ถูกผูกกับ Affiliate ถาวร (Lifetime Binding) |
| **Seller** | ลงสินค้า กำหนด Commission Rate → จ่ายเป็นก้อนเดียว + Platform Fee |
| **Affiliate** | แชร์ลิงก์ขายสินค้า → รับส่วนแบ่งจาก Commission Pool |
| **Team Leader (แม่ทีม)** | ชวน Affiliate เข้าระบบ → รับส่วนแบ่งจาก Pool ของลูกทีมชั้น 1 (Direct Referral) |
| **Admin** | อนุมัติ ตรวจสอบ จัดการถอนเงิน |

### หลักการสำคัญ
- **"Commission Pool ก้อนเดียว"** — Seller กำหนด Commission เท่าไหร่ ระบบนำไปกระจายภายใน Pool นั้น ไม่มีการบวกเพิ่ม
- **Platform Fee แยกต่างหาก** — คิดจากยอดขายเต็ม

---

## 2. Lifetime Customer Binding

- ลูกค้าเข้าผ่าน Affiliate Link ครั้งแรก → ระบบ **ผูก (Bind)** ลูกค้ากับ Affiliate **ถาวรตลอดชีวิต**
- ลูกค้าซื้ออะไรก็ตามในอนาคต → Affiliate เดิมได้ Commission ทุกครั้ง
- **ไม่มีวันหมดอายุ** ไม่ต้องคลิกลิงก์ซ้ำ
- ต่างจาก Buydi เดิมที่ใช้ Cookie 30 วัน

**ข้อจำกัด:** Affiliate รายใหม่อาจหาลูกค้าได้ยากขึ้น เพราะลูกค้าอาจถูกผูกกับคนก่อนหน้าแล้ว

---

## 3. Direct Referral (ทีมชั้น 1 เท่านั้น)

- A ชวน B เข้าระบบ → B เป็น **Direct Referral** ของ A
- B ชวน C เข้าระบบ → C เป็น Direct Referral ของ B
- **C ไม่ใช่ Direct Referral ของ A** (นับเฉพาะชั้นที่ 1)
- ใช้เป็นฐานในการนับโครงสร้างทีมและสิทธิ์ใน Tier / Commission / Leader

---

## 4. Flow การสั่งซื้อ

1. ลูกค้าคลิก Affiliate Link
2. เลือกสินค้า → Checkout + ชำระเงิน
3. **รอพ้นระยะคืนสินค้า** (Return Period)
4. Order = **Complete** → ระบบสร้าง Commission Pool
5. คำนวณยอดสุทธิ (หลังหักส่วนลด) × Rate ที่ Seller กำหนด = Pool
6. ระบบกระจาย Pool เข้า Wallet ของแต่ละคน

### นับเป็น Commission:
- Order Complete (พ้นระยะคืนสินค้าแล้ว) เท่านั้น
- หักส่วนลดแล้ว = ยอดสุทธิ
- Seller กำหนด Rate เอง (ต่อสินค้า)

### ไม่นับ:
- ยกเลิก / คืนสินค้า
- Order ที่ยังไม่พ้นระยะคืนสินค้า
- Order ที่ยังไม่สำเร็จ

## 5. Commission Pool — แบ่ง 2 ก้อน

Seller ตั้ง Commission ก้อนเดียว → ระบบแบ่งเป็น 2 ส่วน + Platform Fee แยก

### สูตรการคำนวณ
```
Commission Pool = ยอดขายสุทธิ × Commission Rate ที่ Seller ตั้ง
ก้อนที่ 1 = 80% ของ Pool (จ่ายทันทีเมื่อ Order Complete)
ก้อนที่ 2 = 20% ของ Pool (หักเข้า Platform สำรองจ่ายรายเดือน/ไตรมาส)
Platform Fee = ยอดขายเต็ม × อัตรา Platform Fee (แยกต่างหาก)
Seller เหลือ = ยอดขาย - Commission Pool - Platform Fee
```

### ตัวอย่างพื้นฐาน
- สินค้าราคา **1,000 บาท**
- Seller ตั้ง Commission **20%** → Pool = **200 บาท**
- Platform Fee **3%** → **30 บาท**
- **ก้อนที่ 1 (80%)** = 160 บาท → จ่ายค่านายหน้า + ค่าลิขสิทธิ์ทันที
- **ก้อนที่ 2 (20%)** = 40 บาท → Platform สำรองจ่ายค่าบริหาร / ส่วนแบ่งยอดขาย / โบนัสไตรมาส
- **Seller จ่ายรวม** = 200 + 30 = **230 บาท**
- **Seller เหลือ** = 1,000 - 230 = **770 บาท**

## 8. รายละเอียดแต่ละประเภทรายได้

### 8.1 ค่านายหน้า (ประเภทที่ 1) — จากก้อนที่ 1
- Affiliate แชร์ลิงก์ → ลูกค้าซื้อ → ได้ส่วนแบ่งจากก้อน 1
- **ได้ทุก Tier** (สมัครฟรีก็ได้)
- **Lifetime Binding** — ลูกค้าซื้อกี่ครั้งก็ได้ทุกครั้ง
- = ก้อน 1 หลังหักค่าลิขสิทธิ์แม่ทีม (ถ้ามี)

**สูตร:**
```
ค่านายหน้า = ก้อน 1 - ค่าลิขสิทธิ์แม่ทีม
           = (80% x Pool) - (% ตาม Tier แม่ทีม x ก้อน 1)
```

### 8.2 ค่าลิขสิทธิ์ (ประเภทที่ 2) — จากก้อนที่ 1
- แม่ทีม (คนชวน Affiliate เข้าระบบ) ได้ส่วนแบ่งจากก้อน 1
- ต้องเป็น **Silver ขึ้นไป** ถึงจะได้
- คิดเป็น % ของก้อน 1 ตาม Tier ของแม่ทีม
- **ถ้าไม่มีแม่ทีม → คนขายได้เต็มก้อน 1**

**ตารางตัวอย่าง (Pool = 200 บาท, ก้อน 1 = 160 บาท):**

| Tier แม่ทีม | % ลิขสิทธิ์ | แม่ทีมได้ | คนขายได้ |
|-------------|------------|----------|---------|
| ไม่มีแม่ทีม | 0% | 0 บาท | **160 บาท** |
| Silver | 10% | 16 บาท | **144 บาท** |
| Gold | 20% | 32 บาท | **128 บาท** |
| Platinum | 25% | 40 บาท | **120 บาท** |
| Wisdom | 25% | 40 บาท | **120 บาท** |

**สำคัญ:** คนขายจริง (Affiliate) ได้ขั้นต่ำ **75% ของก้อน 1** = 120 บาท (เมื่อแม่ทีมเป็น Platinum/Wisdom)

### 8.3 ค่าบริหาร (ประเภทที่ 3) — จากก้อนที่ 2
- จ่ายสิ้นเดือน
- คำนวณจากรายได้ค่านายหน้า + ค่าลิขสิทธิ์ของทีม
- Silver: **5%** | Gold: **10%** | Platinum: **15%** | Wisdom: **20%**

**เงื่อนไข:**
- ยอดส่วนตัว >= 500 บาท/เดือน
- รายได้ทีม >= 1,000 บาท/เดือน

### 8.4 ส่วนแบ่งยอดขาย (ประเภทที่ 4) — จากก้อนที่ 2
- จ่ายสิ้นเดือน
- คำนวณจากยอดรวมค่านายหน้า + ค่าลิขสิทธิ์ทั้งบริษัท
- Gold: **10%** | Platinum: **15%** | Wisdom: **20%**

**เงื่อนไข:**
- ยอดส่วนตัว >= 1,000 บาท/เดือน
- รายได้ทีม >= 12,000 บาท/เดือน

### 8.5 โบนัสรายไตรมาส (ประเภทที่ 5) — จากก้อนที่ 2
- จ่ายทุก 3 เดือน
- คำนวณจากรายได้ค่านายหน้า + ค่าลิขสิทธิ์เฉลี่ย 3 เดือน
- Gold: **25%** | Platinum: **50%** | Wisdom: **100%**

**เงื่อนไข (ต้องผ่านทุกข้อ):**
1. ยอดสั่งซื้อส่วนตัวตามตำแหน่ง **ครบ 3 เดือน**
2. รายได้รวม 3 เดือน >= **60,000 บาท**
3. จำนวน USER สะสม >= **100 คน**

## 9. ตัวอย่างการคำนวณแบบ Step-by-Step

### เคส: สินค้า 1,000 บาท | Commission 20% | แม่ทีม B เป็น Gold | Affiliate C เป็นคนขาย

**Step 1: Seller ตั้งราคาและ Commission**
- ราคาขาย = 1,000 บาท
- Commission Pool = 20% × 1,000 = **200 บาท**
- Platform Fee = 3% × 1,000 = **30 บาท** (แยก)

**Step 2: ก้อนที่ 1 — จ่ายทันที (80% ของ Pool)**
- ก้อน 1 = 80% × 200 = **160 บาท**
- แม่ทีม B เป็น Gold → ค่าลิขสิทธิ์ 20% × 160 = **32 บาท**
- Affiliate C ได้ค่านายหน้า = 160 - 32 = **128 บาท**
- เข้า Wallet ทันทีเมื่อ Order Complete

**Step 3: ก้อนที่ 2 — เข้า Platform สำรอง (20% ของ Pool)**
- ก้อน 2 = 20% × 200 = **40 บาท** → Platform เก็บไว้
- สำรองจ่ายค่าบริหาร (สิ้นเดือน)
- สำรองจ่ายส่วนแบ่งยอดขาย (สิ้นเดือน)
- สำรองจ่ายโบนัสรายไตรมาส (ทุก 3 เดือน)

**สรุปเงินไหลจาก Order นี้:**
| ผู้รับ | จำนวน | หมายเหตุ |
|--------|--------|----------|
| Affiliate C (คนขาย) | **128 บาท** | ค่านายหน้า เข้า Wallet ทันที |
| แม่ทีม B (Gold) | **32 บาท** | ค่าลิขสิทธิ์ เข้า Wallet ทันที |
| Platform สำรอง | **40 บาท** | จ่ายค่าบริหาร/ส่วนแบ่ง/โบนัส ทีหลัง |
| Platform Fee | **30 บาท** | รายได้ Platform |
| **Seller เหลือ** | **770 บาท** | 1,000 - 200 - 30 |

## 10. Platform Fee

- **แยกจาก Commission Pool** — ไม่ได้อยู่ใน Pool
- คิดจาก **ยอดขายเต็ม** (ไม่ใช่ยอดหลังหัก commission)
- อัตราตั้งค่าได้จากหลังบ้าน
- Logic เดียวกับ TikTok / แพลตฟอร์มทั่วไป
- เป็น **รายได้หลักของ Platform**

**สูตร:**
```
Seller เหลือ = ยอดขาย - Commission Pool - Platform Fee
ตัวอย่าง: 1,000 - 200 - 30 = 770 บาท
```

## 11. Wallet & การถอนเงิน

1. เมื่อ Order Complete → ค่านายหน้า + ค่าลิขสิทธิ์ เข้า Wallet ทันที
2. สิ้นเดือน → ค่าบริหาร + ส่วนแบ่งยอดขาย เข้า Wallet (ถ้าผ่านเงื่อนไข)
3. ทุก 3 เดือน → โบนัสไตรมาส เข้า Wallet (ถ้าผ่านเงื่อนไข)
4. ผู้ใช้กด Withdraw → Admin ตรวจสอบ → อนุมัติ → โอนเงินเข้าบัญชี

## 12. เงินที่เหลือจากกองสำรอง (Reserve Policy — ยังต้อง Confirm)

ถ้าไม่มีใครผ่านเงื่อนไขรับค่าบริหาร/ส่วนแบ่ง/โบนัส เงินในก้อนที่ 2 จะทำอย่างไร?

| ทางเลือก | รายละเอียด | ข้อดี |
|---------|-----------|------|
| 1. สะสมต่อ | เงินที่เหลือสะสมไปรอบถัดไป กองโบนัสใหญ่ขึ้นเรื่อยๆ | กองโบนัสโต แรงจูงใจสูง |
| 2. Platform เก็บ | ปิดรอบชัดเจน ไม่สะสม Platform ได้รายได้เพิ่ม | เรียบง่าย + Platform มีรายได้ |
| 3. ผสม (แนะนำ) | 50% สะสมไปรอบถัดไป + 50% Platform เก็บ | สมดุลทั้งสองฝ่าย |

## 13. เปรียบเทียบ Pordeenomad vs BStore

### โมเดลธุรกิจ
| | BStore | Pordeenomad |
|---|--------|------------|
| โมเดล | Platform + ขายสินค้าเอง | Marketplace — Seller ภายนอกลงขาย |
| ควบคุมต้นทุน | 100% | ไม่รู้ต้นทุนสินค้า |
| Platform Fee | ไม่มี (เป็นเจ้าของเอง) | มี (แยกต่างหาก) |
| Commission | บวกเพิ่มหลายชั้นได้ | ต้องใช้หลัก "เงินก้อนเดียว" |

### ใครจ่ายอะไร
| รายการ | BStore | Pordeenomad |
|--------|--------|------------|
| ค่านายหน้า | BStore (กำไรสินค้า) | **Seller (Pool)** |
| ค่าลิขสิทธิ์ | BStore (บวกเพิ่ม) | **Seller (Pool เดียวกัน)** |
| ค่าบริหาร | BStore (กำไร) | **Platform (จากก้อน 2)** |
| ส่วนแบ่งยอดขาย | BStore (ยอดรวม) | **Platform (จากก้อน 2)** |
| โบนัสไตรมาส | BStore (กำไร) | **Platform (จากก้อน 2)** |
| Platform Fee | ไม่มี | **Seller จ่าย (แยก)** |

### โครงสร้าง Commission ต่างกัน
- **BStore = Additive (บวกเพิ่ม):** สินค้า 1,000 บาท Commission 20% → Affiliate ได้ 200 บาท + Referer ได้ 60 บาท (บวกเพิ่ม) = BStore จ่ายรวม 260 บาท
- **Pordeenomad = Single Pool:** สินค้า 1,000 บาท Commission 20% → Pool 200 บาท → แบ่งภายใน Pool (Affiliate 160 + แม่ทีม 40 = 200 บาทเท่าเดิม) Seller จ่ายแค่ที่ตั้งไว้

## 14. ประเด็นที่ยังต้อง Confirm

2. **% ค่าลิขสิทธิ์ตาม Tier** — Silver 10% / Gold 20% / Plat-Wisdom 25% คนขายได้พอจูงใจไหม?
3. **เงินเหลือจากก้อน 2** — สะสม / Platform เก็บ / ผสม?
4. **อัตรา Platform Fee สุดท้าย** — 3% เหมาะสมหรือไม่?
5. **กรณี Cross-Tier** — แม่ทีมกับลูกทีมคนละ Tier ต้องยกตัวอย่างหลายเคสเพื่อ confirm logic

## 15. การแสดงผล Commission (UX)

- แสดง Commission อิงจาก **ยอดขายเต็ม** เหมือนแพลตฟอร์มทั่วไป
- ตัวอย่าง: "Commission 10%" = ลูกค้าซื้อ 1,000 → Affiliate ได้ 100 บาท
- การหัก Platform Fee / แบ่งแม่ทีม เป็นกระบวนการภายหลัง
- **ไม่ควร:** หัก Platform Fee ก่อนแล้วค่อยคำนวณ Commission (ทำให้ตัวเลขไม่ตรงกับความคาดหวัง)
