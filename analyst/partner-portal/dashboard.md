# BA Specification: Partner Portal — แดชบอร์ด (Dashboard)

**Project:** PordeeConnect Partner Portal
**Module:** Dashboard
**Version:** 1.0
**Date:** 2026-05-06
**UI Reference:** `partner-portal.html` → `#page-dashboard`

---

## Objective

แสดงภาพรวมผลประกอบการของ Partner ในเดือนปัจจุบัน ทั้งรายได้ ยอดขาย สถานะตำแหน่ง และภาพรวมทีม — เป็นหน้าแรกที่ Partner เห็นเมื่อเข้าระบบ เพื่อให้ประเมินสถานการณ์ได้ทันทีโดยไม่ต้องเจาะไปหน้าอื่น

---

## Scope

### In Scope
- แสดง Stats Cards สรุป KPI 4 ตัว (ทุกคนเห็นเหมือนกัน)
- แสดงตารางสรุปรายได้เดือนปัจจุบันแยกตามประเภท (settled only)
- แสดง Quick View ทีมล่าสุด (ถ้ายังไม่มีลูกทีม → empty state)
- แสดง Banner โปรโมชั่นพิเศษ / แคมเปญที่กำลังดำเนินการ พร้อม progress

### Out of Scope
- การแก้ไข / จัดการรายได้ (ดำเนินการในหน้า "รายได้ของฉัน")
- การจัดการสมาชิกในทีม (ดำเนินการในหน้า "ทีมของฉัน")
- การสมัคร / จัดการแคมเปญ (ดำเนินการในหน้า "แคมเปญ")
- การดูรายละเอียดออเดอร์ (ดำเนินการในหน้า "สินค้าที่ขายได้")
- Notification center / การแจ้งเตือน
- การตั้งค่าโปรไฟล์ Partner

---

## User Roles

**Partner ทุกคนเห็น UI เหมือนกัน** — ไม่มีการแยก Leader / Member ในหน้าแดชบอร์ด

ทุกคนจะเห็น Stats Cards 4 การ์ด + ตารางรายได้ + ทีมล่าสุด + โปรโมชั่น เหมือนกันหมด ความแตกต่างอยู่ที่ **ข้อมูล**:
- Partner ที่มีลูกทีม → การ์ดสมาชิกแสดงจำนวน, panel ทีมแสดงรายชื่อ
- Partner ที่ยังไม่มีลูกทีม → การ์ดสมาชิกแสดง "0 คน", panel ทีมแสดง empty state "ยังไม่มีสมาชิกในทีม"

---

## Precondition

- **PC-1:** ผู้ใช้ต้อง login เข้าระบบ PordeeConnect แล้ว และมี session ที่ valid
- **PC-2:** ระบบต้องดึงข้อมูลรายได้ ยอดขาย และตำแหน่งของเดือนปัจจุบันได้สำเร็จ

**กรณี Precondition ไม่ครบ:**

| กรณี | พฤติกรรม |
|------|---------|
| ยังไม่ login | Redirect ไปหน้า login |
| API ดึงข้อมูลล้มเหลว | แสดง skeleton loading → หลัง timeout แสดง error state "ไม่สามารถโหลดข้อมูลได้ กรุณาลองใหม่" พร้อมปุ่ม retry |
| Partner ใหม่ ยังไม่มีข้อมูล | แสดง empty state — Stats Cards แสดงค่า ฿0 / 0 คน, ตารางรายได้แสดง "ยังไม่มีรายได้ในเดือนนี้", โปรโมชั่นแสดงตามปกติ |

## Business Rules

| # | Rule | รายละเอียด | ตัวอย่าง |
|---|------|-----------|---------|
| BR-1 | **ข้อมูลเป็นเดือนปัจจุบัน** | Stats Cards และตารางรายได้แสดงข้อมูลของเดือนปัจจุบันเท่านั้น (1–วันปัจจุบันของเดือน) | เดือน พ.ค. 2569 แสดงข้อมูล 1–6 พ.ค. |
| BR-2 | **% เปลี่ยนแปลงเทียบเดือนก่อน** | คำนวณจาก `((เดือนนี้ - เดือนก่อน) / เดือนก่อน) × 100` แสดงทศนิยม 1 ตำแหน่ง | ฿17,150 vs ฿15,244 = ↑ 12.5% |
| BR-3 | **ตารางแสดงเฉพาะยอด settled** | ตารางสรุปรายได้แสดงเฉพาะยอดที่ "จ่ายแล้ว" (order complete + พ้นระยะคืนสินค้า + เข้ากระเป๋าถอนได้) — ไม่มีคอลัมน์สถานะ แต่มี tooltip ⓘ ที่หัวตารางอธิบายนิยาม | หัวตาราง hover → "ยอดที่แสดงคือรายได้จากออเดอร์ที่สำเร็จและพ้นระยะคืนสินค้าแล้ว " |
| BR-4 | **การ์ด "สมาชิกในทีม" แสดงทุกคน** | แสดงจำนวนลูกทีมทุกชั้น ถ้ายังไม่มีลูกทีม → แสดง "0 คน" | Partner ใหม่เห็น 0 คน, Partner ที่มีทีมเห็นจำนวนจริง |
| BR-5 | **Panel "ทีมล่าสุด" แสดงทุกคน** | แสดงลูกทีม top 4 เสมอ ถ้ายังไม่มีลูกทีม → แสดง empty state "ยังไม่มีสมาชิกในทีม" | ไม่ซ่อน panel — layout 2 คอลัมน์เสมอ |
| BR-6 | **Progress ตำแหน่ง** | คำนวณจาก `ยอดขายสะสม / เป้าหมายตำแหน่งถัดไป × 100` แสดงเป็น progress bar | Gold → Platinum ต้อง ฿50,000, ทำได้ ฿45,200 = 90.4% (เหลือ ฿4,800) |
| BR-7 | **ตำแหน่งสูงสุด (Wisdom)** | ถ้าอยู่ตำแหน่ง Wisdom แล้ว → แสดง "ตำแหน่งสูงสุดแล้ว" แทน progress bar | ไม่แสดง "เหลืออีก ฿x ถึง..." |
| BR-8 | **ทีมล่าสุด แสดงสูงสุด 4 คน** | เรียงตามยอดขายสูงสุดของเดือนปัจจุบัน จาก direct downline (ลูกทีมตรง) | แสดง avatar, ชื่อ, ประเภท, ยอดขาย, คอมมิชชั่น, ตำแหน่ง |
| BR-9 | **โปรโมชั่น Banner** | แสดงแคมเปญที่ active โดยเลือกแคมเปญที่ใกล้หมดเวลาที่สุด 1 รายการ | ถ้ามี 2 แคมเปญ active (ญี่ปุ่น เหลือ 42 วัน vs ฮ่องกง เหลือ 224 วัน) → แสดงญี่ปุ่น |
| BR-10 | **Progress แคมเปญ** | คำนวณจาก `ยอดสะสม / เป้าหมาย × 100` แสดงเป็น % จำนวนเต็ม | ฿68,000 / ฿100,000 = 68% |
| BR-11 | **Mini chart 7 แท่ง** | แสดงรายได้รวม 7 วันล่าสุด (หรือ 7 สัปดาห์ล่าสุด) ความสูงแท่งเป็น relative กับค่าสูงสุด | แท่งสุดท้าย (วันปัจจุบัน/สัปดาห์ปัจจุบัน) แสดงสี primary เข้ม |
| BR-12 | **ลำดับตำแหน่ง (Tier)** | Starter → Silver → Gold → Platinum → Wisdom | ใช้คำนวณ progress ในการ์ดตำแหน่ง และแสดง rank badge |
| BR-13 | **Tooltip เงื่อนไขรายได้** | Team Management Fee และ Sales Sharing แสดง tooltip เมื่อ hover ที่ชื่อประเภท — แสดงเงื่อนไขคุณสมบัติและสถานะผ่าน/ไม่ผ่าน | TMF: "ยอดขาย ≥ ฿1,000 + คอมมิชชั่น ≥ ฿1,000 → ✅ ผ่าน" |
| BR-14 | **Click-through จากการ์ด** | Stats Cards และ Banner เป็น clickable — กดแล้ว navigate ไปหน้าที่เกี่ยวข้อง | การ์ดรายได้→หน้ารายได้, ยอดขาย→สินค้า, สมาชิก→ทีม, Banner→แคมเปญ |

## Main Flow

### Flow 1: โหลดหน้าแดชบอร์ด (Happy Path)

```
1. ผู้ใช้เข้าระบบสำเร็จ หรือกดเมนู "แดชบอร์ด"
2. ระบบดึงข้อมูลจาก API พร้อมกัน:
   ├─ 2.1 ข้อมูลรายได้รวมเดือนนี้ + % เปลี่ยนแปลง + mini chart data
   ├─ 2.2 ข้อมูลยอดขายส่วนตัว + % เปลี่ยนแปลง
   ├─ 2.3 ข้อมูลตำแหน่งปัจจุบัน + progress ไปตำแหน่งถัดไป
   ├─ 2.4 ข้อมูลรายได้แยกประเภท (settled only)
   ├─ 2.5 ข้อมูลจำนวนสมาชิก + สมาชิกใหม่
   ├─ 2.6 ข้อมูลทีมล่าสุด top 4 (ถ้ามี)
   └─ 2.7 ข้อมูลแคมเปญที่ active
3. ระบบ render หน้าจอ (ทุกคนเห็น UI เหมือนกัน):
   ├─ 3.1 แสดง Stats Cards 4 การ์ด
   ├─ 3.2 แสดงตารางสรุปรายได้ (BR-3)
   ├─ 3.3 แสดง panel ทีมล่าสุด หรือ empty state (BR-5, BR-8)
   ├─ 3.4 แสดง Banner โปรโมชั่น (BR-9, BR-10)
   └─ 3.5 Mini chart แสดง 7 แท่ง (BR-11)
```

### Flow 2: กดดูทีมทั้งหมด

```
1. ผู้ใช้กดปุ่ม "ดูทั้งหมด" ที่ panel ทีมล่าสุด
2. ระบบ navigate ไปหน้า "ทีมของฉัน" (#page-team)
3. อัปเดต sidebar active state ไปที่เมนู "ทีมของฉัน"
```

### Flow 3: Error — API ล้มเหลว

```
1. ระบบพยายามดึงข้อมูลจาก API
2. API respond error หรือ timeout
3. แสดง skeleton shimmer ระหว่างรอ (สูงสุด 10 วินาที)
4. หลัง timeout:
   ├─ ส่วนที่โหลดสำเร็จ → แสดงข้อมูลปกติ
   └─ ส่วนที่ล้มเหลว → แสดง inline error
       "ไม่สามารถโหลดข้อมูลได้" + ปุ่ม "ลองใหม่"
5. กด "ลองใหม่" → เรียก API เฉพาะส่วนที่ fail ใหม่
```

## Data Requirements

> **หมายเหตุสำคัญ:** ยอดทั้งหมดที่แสดงในแดชบอร์ดเป็น **settled amount** — คือยอดจากออเดอร์ที่ complete แล้วและพ้นระยะเวลาคืนสินค้าแล้วเท่านั้น ยอดที่ยังอยู่ระหว่างจัดส่ง / รอยืนยัน / อยู่ในระยะคืนสินค้า จะไม่นับรวม

---

### 1. การ์ด: รายได้รวมเดือนนี้

| Field | Type | ที่มา / สูตรคำนวณ |
|-------|------|-------------------|
| `total_income` | Number (฿) | **ผลรวมรายได้ทุกประเภทที่ settled** ของเดือนที่เลือก = `affiliate_commission` + `royalty` + `team_mgmt_fee` + `sales_sharing` |
| `total_income_prev_month` | Number (฿) | ผลรวมรายได้ทุกประเภทที่ settled ของ **เดือนก่อนหน้า** (ใช้คำนวณ % เปลี่ยนแปลง) |
| `total_income_change_pct` | Number (%) | `((total_income - total_income_prev_month) / total_income_prev_month) × 100` · ทศนิยม 1 ตำแหน่ง · ถ้าเดือนก่อน = 0 → แสดง "—" |
| `income_chart_data` | Array[7] of Number | รายได้รวมต่อวัน ย้อนหลัง 7 วันล่าสุด (ใช้แสดง mini bar chart) |

**ตัวอย่าง:**
```
affiliate_commission = ฿8,200
royalty              = ฿4,500
team_mgmt_fee        = ฿2,350
sales_sharing        = ฿2,100
─────────────────────────────
total_income         = ฿17,150

เดือนก่อน            = ฿15,244
change_pct           = ((17,150 - 15,244) / 15,244) × 100 = ↑ 12.5%
```

---

### 2. การ์ด: ยอดขายส่วนตัว

| Field | Type | ที่มา / สูตรคำนวณ |
|-------|------|-------------------|
| `personal_sales` | Number (฿) | **ยอดขายจากออเดอร์ที่ Partner คนนี้เป็นคนขาย** (ทั้งขายให้คนอื่น + ซื้อใช้เอง) · นับเฉพาะออเดอร์ที่ settled |
| `personal_sales_prev_month` | Number (฿) | ยอดขายส่วนตัวของเดือนก่อนหน้า |
| `personal_sales_change_pct` | Number (%) | `((personal_sales - personal_sales_prev_month) / personal_sales_prev_month) × 100` |

**ที่มาของยอดขาย:**
```
ยอดขายส่วนตัว = SUM(order.total_price)
  WHERE order.seller_partner_id = current_partner
    AND order.status = 'complete'
    AND order.return_period_passed = true
    AND order.created_at BETWEEN เดือนที่เลือก
```

**สิ่งที่นับรวม:**
- ออเดอร์ที่ขายให้ลูกค้าทั่วไปผ่าน affiliate link
- ออเดอร์ที่ Partner ซื้อใช้เอง

**สิ่งที่ไม่นับ:**
- ยอดขายของลูกทีม (แม้จะเป็น downline)
- ออเดอร์ที่ยกเลิก / คืนเงิน / ยังอยู่ในระยะคืนสินค้า

**ตัวอย่าง:**
```
Partner สมหญิง ขายเซรั่ม 16 ชิ้น + ครีมกันแดด 14 ชิ้น + อื่นๆ
= ฿45,200 (เฉพาะ settled orders)
```

### 5. ตารางสรุปรายได้ (4 ประเภท)

> ตารางนี้แสดงเฉพาะยอด **settled** ของเดือนที่เลือก พร้อม month picker ให้ดูเดือนย้อนหลังได้

#### 5.1 ค่าคอมมิชชั่น Affiliate

| Field | สูตร |
|-------|------|
| `affiliate_commission` | `SUM(order.total_price × order.commission_rate)` เฉพาะออเดอร์ที่ Partner คนนี้เป็นคนขาย + settled |

```
ที่มา:    ออเดอร์ที่ลูกค้าสั่งผ่าน affiliate link ของ Partner
อัตรา:    ผู้ขาย (Seller) กำหนดเอง 5-20% ต่อสินค้า
ตัวอย่าง: เซรั่ม ฿890 × 10% × 10 ชิ้น = ฿890
          ครีมกันแดด ฿450 × 10% × 6 ชิ้น = ฿270
          ... รวม = ฿8,200
```

#### 5.2 Royalty / Licensing Fee (2%)

| Field | สูตร |
|-------|------|
| `royalty` | คำนวณแบบ **cascade** ตามตำแหน่ง จากรายได้ค่าคอมมิชชั่นของลูกทีม |

```
ที่มา:    2% จากยอดขายของสินค้าที่ลูกทีมขาย (คำนวณแบบ cascade ตามตำแหน่ง)
วิธีคำนวณ: ขึ้นอยู่กับตำแหน่ง — Gold ได้รับ 30% ของค่า royalty ขั้นก่อนหน้า
ตัวอย่าง: ลูกทีมขายรวม ฿75,000
          royalty pool = ฿75,000 × 2% = ฿1,500
          Gold share = cascade calculation = ฿4,500
```

> **หมายเหตุ:** สูตร cascade ต้องกำหนดรายละเอียดใน PRD — ระบุว่าแต่ละ tier ได้กี่ % และคำนวณจากชั้นไหนถึงชั้นไหน

#### 5.3 Team Management Fee (1%)

| Field | สูตร |
|-------|------|
| `team_mgmt_fee` | `(SUM downline commissions ชั้น 1-3) × 1% × tier_share_pct` |

```
ที่มา:    คำนวณรายคน — นำค่าคอมมิชชั่น affiliate ของลูกทีมชั้น 1, 2, 3 ของ Partner คนนั้นมารวมกัน
          → คิด 1% จากยอดรวม → คูณ % ตามตำแหน่ง (Tier) ของ Partner
Pool:     SUM(ค่าคอมมิชชั่น affiliate ลูกทีมชั้น 1 + ชั้น 2 + ชั้น 3) × 1%
แบ่ง:     Partner ได้ = Pool × tier_share_pct ของตำแหน่งตัวเอง
          (Silver 5% / Gold 10% / Platinum 15% / Wisdom 20%)

ตัวอย่าง: Partner สมหญิง (Gold) มีลูกทีม:
          ชั้น 1 (10 คน): ค่าคอมรวม = ฿50,000
          ชั้น 2 (20 คน): ค่าคอมรวม = ฿30,000
          ชั้น 3 (20 คน): ค่าคอมรวม = ฿20,000
          ─────────────────────────────────────
          รวมค่าคอมลูกทีม 3 ชั้น = ฿100,000
          Pool 1% = ฿100,000 × 1% = ฿1,000
          Gold share (10%) = ฿1,000 × 10% = ฿100

หมายเหตุ: ทุกคนที่ผ่านเงื่อนไขจะได้ค่าบริหารทีมแบบนี้ คำนวณจาก downline ของตัวเอง 3 ชั้นลงไป

เงื่อนไข (tooltip): ยอดขายส่วนตัว ≥ ฿1,000/เดือน AND คอมมิชชั่น ≥ ฿1,000/เดือน
สถานะ:             ✅ ผ่าน (ยอดขาย ฿45,200 ≥ ฿1,000 ✓ · คอม ฿8,200 ≥ ฿1,000 ✓)
```

#### 5.4 Sales Sharing (1%)

| Field | สูตร |
|-------|------|
| `sales_sharing` | `pool × tier_share_pct / qualified_count` |

```
ที่มา:    1% ของยอดขายรวมทั้งบริษัท → รวมเป็น pool → แบ่งตามตำแหน่ง → หารจำนวนคนที่ผ่านเกณฑ์
Pool:     ยอดขายรวมทั้งบริษัท × 1%
แบ่ง:     Gold pool ได้ 10% ของ pool ทั้งหมด
หาร:      จำนวนคนที่ตำแหน่ง Gold + ผ่านเกณฑ์
ตัวอย่าง: ยอดขายบริษัท = ฿3,150,000
          pool = ฿3,150,000 × 1% = ฿31,500
          Gold pool = ฿31,500 × 10% = ฿3,150
          ผู้ผ่านเกณฑ์ Gold = 15 คน (ทั้งนี้ผ่านเกณฑ์จริงหรือไม่ขึ้นกับเงื่อนไขด้านล่าง)
          ต่อคน = ฿3,150 / 15 = ฿210... (mock ใช้ ฿2,100)

เงื่อนไข (tooltip): ยอดขายส่วนตัว ≥ ฿1,000/เดือน AND คอมมิชชั่น ≥ ฿10,000/เดือน
สถานะ:             ❌ ยังไม่ผ่าน (คอม ฿8,200 < ฿10,000 ✗)
```

> **หมายเหตุ:** แม้ยังไม่ผ่านเกณฑ์ ก็ยังแสดงยอดในตาราง เพราะเป็นยอดที่ "คำนวณไว้แล้ว" รอจ่ายเมื่อผ่านเกณฑ์ — ต้องตกลงกับ business ว่าจะแสดงยอดที่ยังไม่ผ่านเกณฑ์หรือไม่

#### สรุปยอดรวม

```
total_income = affiliate_commission + royalty + team_mgmt_fee + sales_sharing
             = ฿8,200 + ฿4,500 + ฿2,350 + ฿2,100
             = ฿17,150
```

## Assumptions & Constraints

### Assumptions
- **A-1:** ข้อมูลรายได้และยอดขายจะถูกอัปเดตแบบ near real-time (ไม่เกิน 5 นาที delay)
- **A-2:** ระบบตำแหน่ง (Tier) มีเกณฑ์ที่ชัดเจนและคงที่ — ถ้าเกณฑ์เปลี่ยน ต้องอัปเดต API response
- **A-3:** ไม่มีการแยก role Leader/Member ในหน้าแดชบอร์ด — ทุกคนเห็น UI เหมือนกัน ความแตกต่างอยู่ที่ข้อมูล (มีลูกทีมหรือยัง)
- **A-4:** โปรโมชั่น Banner สามารถมีได้หลายแคมเปญพร้อมกัน — แต่ในหน้าแดชบอร์ดแสดงเฉพาะแคมเปญที่ใกล้หมดเวลาที่สุด 1 รายการ
- **A-5:** ยอดเงินทั้งหมดเป็นสกุลเงินบาท (THB) — ไม่รองรับ multi-currency

### Constraints
- **C-1:** Mini chart ต้องการข้อมูลย้อนหลังอย่างน้อย 7 จุด — ถ้ามีน้อยกว่า 7 วัน (เช่น Partner ใหม่) ให้แสดงเท่าที่มีและเติม 0 ส่วนที่ขาด

## BA Note

1. **Demo Toolbar ใน Mockup ไม่ใช่ UI จริง:** Topbar ใน mockup มี demo toolbar (ปกติ / Loading / Empty) สำหรับให้ทีมดู UI แต่ละ state — **ไม่ต้อง implement ใน production** ผู้ใช้จะไม่เห็นแถบนี้ เช่นเดียวกับ role switcher ที่เคยมีก่อนหน้า (ตอนนี้เอาออกแล้ว) ส่วน role ถูกกำหนดจาก backend ตั้งแต่ login

2. ~~**สถานะ "รอรอบ Qx":**~~ **[ตัดออก]** ลบประเภทรายได้ Special Bonus และสถานะ "รอรอบ Qx" ออกทั้งหมด — ระบบมีเพียง 2 สถานะ: จ่ายแล้ว / รอจ่าย (**อัปเดตแล้วทั้ง mockup + spec**)

3. **โปรโมชั่น Banner:** **[ตัดสินใจแล้ว]** แสดงแคมเปญที่ใกล้หมดเวลาที่สุด 1 รายการ — เช่น ญี่ปุ่น (เหลือ 42 วัน) แสดงก่อนฮ่องกง (เหลือ 224 วัน) (**อัปเดตแล้วใน BR-9**)

4. **Tooltip เงื่อนไขรายได้:** **[ตัดสินใจแล้ว]** เพิ่ม tooltip แสดงเงื่อนไขคุณสมบัติของ Team Management Fee และ Sales Sharing ในตารางรายได้บนแดชบอร์ด — hover ที่ชื่อประเภทจะแสดงเงื่อนไข + สถานะผ่าน/ไม่ผ่าน (**อัปเดตแล้วทั้ง mockup + spec — BR-13, AC-18**)

5. **Empty State สำคัญ:** Partner ใหม่ที่เพิ่งสมัครจะเห็นแดชบอร์ดที่ค่าเป็น 0 ทั้งหมด — ควรออกแบบ empty state ที่เป็นมิตร เช่น แสดง onboarding tips หรือ quick action buttons ("สร้างลิงก์แรก", "เลือกสินค้าที่จะขาย")

6. **Performance:** เนื่องจากหน้าแดชบอร์ดต้องเรียก API หลายตัวพร้อมกัน — ควรใช้ parallel API calls และ partial rendering (แสดงส่วนที่พร้อมก่อน ไม่ต้องรอทั้งหมด)

7. **Click-through จากการ์ด:** **[ตัดสินใจแล้ว]** เพิ่ม click-through จากการ์ดและ banner ไปหน้าที่เกี่ยวข้อง (**อัปเดตแล้วทั้ง mockup + spec — BR-14, AC-19**):
   - การ์ด "รายได้รวม" → หน้ารายได้ของฉัน
   - การ์ด "ยอดขายส่วนตัว" → หน้าสินค้าที่ขายได้
   - การ์ด "สมาชิกในทีม" → หน้าทีมของฉัน
   - Banner โปรโมชั่น → หน้าแคมเปญ
